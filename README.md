# ⛳ ゴルフ弾道トラッカー

ゴルフのティーショットを **後ろ（打者の背後）から撮影した動画** を読み込み、
ボールの **飛球軌道ライン** を映像に重ねて表示・書き出し・共有できる Web アプリです。

- **完全フロントエンド**（サーバー不要）。動画は端末のブラウザ内だけで処理され、外部に送信されません。
- **スマホ（iPhone / Android）のブラウザ** で動作します。
- 無料の静的ホスティング（GitHub Pages など）で公開できます。

## 使い方

1. **動画を選ぶ** — 飛球方向の後ろから撮ったティーショットの動画を選択します。
   きれいな軌道を出すには **スロー撮影（120/240fps）推奨** です。
2. **ボール位置を指定（任意・推奨）** — スライダーでインパクト直後あたりまで進め、
   ボールをタップして開始位置を教えると検出精度が上がります。
3. **解析する** — 「解析して軌道を描く」を押すと、コマ送りで動画を解析し、軌道ラインを描きます。
   うまく出ないときは **検出感度** を上げ下げして再解析してください。
4. **書き出し・共有** — 軌道入りの動画を元の速さで書き出し、ダウンロード／他アプリへ共有できます。

## なぜスロー撮影が推奨なのか

ティーショットの初速は秒速約 55m（時速 200km 超）。通常撮影（30fps）ではボールが
1〜2 コマしか写らず、軌道を描くための情報が足りません。撮影時に多くのコマ数で
記録する **撮影スロー（240fps など）** が最も確実です（※後から再生をスローにしても
情報は増えません）。本アプリは通常動画も読み込めますが、その場合は精度が落ちます。

## うまく軌道を出すコツ

- できれば **スロー撮影（120/240fps）** を使う
- カメラは固定（三脚や置き撮り）し、手ブレを抑える
- 背景はできるだけ **空や単色**、順光で撮る（逆光を避ける）
- ボールが画面内を長く飛ぶよう、やや引き気味のアングルで撮る

## 仕組み（技術概要）

- **検出**: コマ間の差分（動いた明るい領域）を抽出し、連結成分（ブロブ）として
  ラベリング。直前位置と速度から次の位置を予測し、最もボールらしいブロブを追跡します
  （`js/tracker.js`）。
- **軌道整形**: 進行方向に矛盾する誤検出を除去し、滑らかな曲線として描画します。
- **書き出し**: `canvas.captureStream()` + `MediaRecorder` で軌道を重ねた映像を録画。
  iOS/Safari では mp4、対応がなければ webm で出力します（`js/exporter.js`）。
- フレーム走査は `requestVideoFrameCallback` を使用し、高 fps の動画でもコマ落ちしない
  よう低速再生でスキャンします。

## ローカルで動かす

ES モジュールを使っているため、`file://` で直接開くのではなく簡易サーバー経由で開きます。

```bash
# 例: Python の簡易サーバー
python3 -m http.server 8000
# ブラウザで http://localhost:8000 を開く
```

## 公開する（GitHub Pages）

1. このリポジトリを GitHub に push
2. Settings → Pages → Source を `main`（または対象ブランチ）/ root に設定
3. 発行された URL をスマホのブラウザで開く

## ファイル構成

```
index.html             画面
css/styles.css         スタイル
js/tracker.js          ボール検出・軌道追跡
js/exporter.js         軌道入り動画の書き出し
js/app.js              画面とロジックの接続
manifest.webmanifest   PWA 設定（ホーム画面に追加）
icons/                 アプリアイコン・favicon
```

ホーム画面に追加（PWA）すると、アプリアイコンから全画面で起動できます。

## 制限・今後の改善余地

- 精度は背景・光・撮影角度に左右されます（MVP のため、まず条件の良い動画で確実に動くことを優先）。
- 現状は **2D の軌道ライン** 表示です。飛距離・打ち出し角の推定や 3D 表示は今後の拡張候補です。
- 書き出し動画には音声は含まれません。
